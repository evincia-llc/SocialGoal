# Session close

1. **`ai-context/tasks.md`:** update Phase / Current sprint / Branch state;
   rewrite the "Now" list so the next session's first action is unambiguous;
   append a session-log entry (2-4 lines: what moved, what surprised, what's
   next). Newest first.
2. **`ai-context/backlog.md`:** flip any statuses that changed
   (`todo`/`active`/`done`/`blocked(reason)`). Never edit scope here.
3. **`ai-context/journal.md`:** for each problem or roadblock hit this session
   that isn't already logged, add an entry per the template. Zero new entries is
   acceptable only if the session genuinely hit no friction -- do not backfill
   from memory in later sessions.
4. **`ai-context/decisions.md`:** record any decision made in-session; list any
   newly OPEN decisions in the close-out summary so the operator sees them.
5. **Dangling work check:** report uncommitted changes and their branch, plus any
   running background tasks, so nothing is silently lost between sessions.
