# SBOM / SCA baseline -- Sprint 1 (2026-07-23)

The recorded known-vulnerable set for the legacy solution at containment
(R-008 baseline). Nothing here is remediated in Sprint 1 by design: the epic
remediates the critical subset that can move safely in Sprint 4, replaces the
web/auth stack wholesale in Phase 2, and consolidates front-end assets in
Sprint 12. Sprint 13's exit gate is a clean scan.

Artifacts in this folder:

- `sbom-nuget.cdx.json` -- CycloneDX SBOM of all 41 NuGet packages
  (generated by the `CycloneDX` dotnet tool against `source/SocialGoal.sln`).
- `nuget-audit-baseline.txt` -- accepted NuGet audit findings
  (`NU1902`/`NU1903` from `nuget restore`); the security workflow fails when a
  finding appears that is not in this list.
- `retire-baseline.txt` -- accepted retire.js findings over the vendored
  browser assets (`file|component|version`); same new-finding gate.

## Vulnerable vendored browser assets (retire.js, `Scripts/`)

The script museum: five distinct jQuery versions, two jQuery UI versions,
handlebars RC, and knockout 2.x, all vendored loose in
`source/SocialGoal/Scripts/`. `Content/` scanned clean. Highest-severity
items: handlebars 1.0.rc.1 (multiple High: prototype pollution, arbitrary
code execution) and every jQuery 1.x file (XSS family, all EOL).

| Component | Version(s) | Files |
|---|---|---|
| handlebars | 1.0.rc.1 | 1 |
| jquery | 1.2.6, 1.6.2, 1.7.1, 1.7.2 (incl. `jquery-latest.js`) | 10 |
| jquery-ui (+dialog) | 1.8.20, 1.8.21 | 3 |
| knockout | 2.0.0, 2.1.0 | 4 |

Note the runtime exposure is narrower than the file list: bundles serve
jQuery 1.7.2 (`BundleConfig.cs`) even though NuGet packages 1.10.2 -- the
vendored copies are what actually ships to browsers. Full file-level detail in
`retire-baseline.txt`. Disposition: Sprint 12 deletes the museum and leaves one
current copy of each surviving dependency.

## Gate semantics (scan of record)

`.github/workflows/security.yml` is the PR security scan of record (repo
`CLAUDE.md` governance):

1. **Secret scan** (gitleaks, full history) -- must be clean; no baseline.
2. **NuGet audit** -- fails on any finding not in `nuget-audit-baseline.txt`.
3. **retire.js** -- fails on any `file|component|version` not in
   `retire-baseline.txt`.

Shrinking the baselines (remediation) is always allowed; growing them requires
a decision recorded in `ai-context/decisions.md`.
